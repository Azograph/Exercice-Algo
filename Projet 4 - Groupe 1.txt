Algorithme VerifierDisponibilite
Entrées :
	nomRecherche (chaîne de caractères)
	listeProduits (tableau de produits trié alphabétiquement)
Sorties :
	message indiquant disponibilité et quantité
Début
	debut ← 0
	fin ← taille(listeProduits) - 1
	trouve ← FAUX
	Tant que debut ≤ fin ET trouve = FAUX faire
 	   milieu ← (debut + fin) DIV 2
    	Si listeProduits[milieu].nom = nomRecherche alors
        	trouve ← VRAI
    	Sinon si nomRecherche < listeProduits[milieu].nom alors
        	fin ← milieu - 1
    	Sinon
        	debut ← milieu + 1
    	FinSi
	FinTantQue
 
	Si trouve = VRAI alors
    	Si listeProduits[milieu].quantite > 0 alors
        	Afficher "Produit disponible"
        	Afficher "Quantité :", listeProduits[milieu].quantite
    	Sinon
        	Afficher "Produit en rupture de stock"
    	FinSi
	Sinon
   	 Afficher "Produit introuvable"
	FinSi

TRACAGE verifierDisponibilite(tab, aliment = ‘lentilles corail’) :
debut = 0
fin = 7 - 1  = 6
trouve = faux
Début tant que debut ≤ fin ET trouve = FAUX : 
TOUR 1 : 
debut <= fin, trouve = faux donc :
milieu = (0 + 6 ) / 2 = 3
On compare tab[milieu][0] = ‘Haricot vert’ avec aliment = ‘lentilles corail’
On a ‘Haricot vert’ < ‘lentilles corail’ alphabétiquement, donc : 
debut = milieu + 1 = 4

TOUR 2 : 
debut <= fin, trouve = faux donc : 
milieu = (4 + 6) / 2 = 5
On compare tab[5][0] = ‘riz basmati’ avec aliment = ‘lentilles corail’
On a ‘Riz basmati’ > ‘lentilles corail’ alphabétiquement, donc : 
fin = milieu - 1 = 4

TOUR 3 : 
debut = fin = 4, trouve = faux donc : 
milieu = 8 / 2 = 4
On compare tab[4][0] = ‘lentilles corail’ avec ‘lentilles corail’
Ils sont égaux donc : trouve = vrai

TOUR 4 : 
debut = fin = 4, trouve = vrai donc on sort du TANT QUE
 
On a tab[milieu][0] = 0 donc : 
On affiche "Produit en rupture de stock"

Situation : Chaque médicament a une date de péremption. 
Chaque matin, la pharmacie veut être alertée des médicaments périmés ou qui le seront dans moins de 30 jours. 
1. Décrire l'algorithme qui parcourt le stock et identifie les médicaments à retirer ou surveiller. 
2. Comment distinguer "déjà périmé" et "périme bientôt" dans votre pseudocode ? 
3. Créer un tableau de 7 médicaments, accompagné de leur date de péremption, en suivant le modèle suivant :
 [ [Amoxicilline, 03/04/2026] , [Ozempic, 07/07/2026] , [Paracétamol, 13/05/2026] ] 
Effectuer le traçage de l’algorithme en prenant comme date du jour le 16/04/2026 

FONCTION alertePeremption(tableau)
    dateDuJour <- date actuelle de l'ordinateur

    POUR i de 0 à longueur(tableau) -1
	anneeMin <- année de tableau[i][1]
	moisMin <- mois de tableau[i][1]
	jourMin <- jour de tableau[i][1]
	30Jours <- 30 derniers jours par rapport date actuelle de l'ordinateur
	SI anneeMin + moisMin + jourMin > dateDuJour
	    AFFICHER 'Déjà périmé' + tableau[i][0]
	SINON SI anneeMin + moisMin + jourMin =< 30Jours 
	    AFFICHER 'Périme bientôt' + tableau[i][0]
	FIN SI	 
    FIN POUR
FIN FONCTION

FONCTION triPeremption(tableau)
	Pour x allant de 0 à taille tableau – 2
	indexMin = x			   //0
	anneeMin <- année de tableau[x][1] // 2026
	moisMin <- mois de tableau[x][1]   // 04
	jourMin <- jour de tableau[x][1]  // 03
		Pour i allant de x + 1 à taille tableau – 1
			annee <- année de tableau[i][1] // 2026
			mois <- mois de tableau[i][1]  //07
			jour <- jour de tableau[i][1] //07
				SI anneeMin > annee
					indexMin <- i 
				SINON SI anneeMin = annee 
					SI moisMin > mois  
						indexMin <- i  
					SINON SI moisMin = mois 
						SI jourMin > jour
							indexMin <- i 
						FIN SI
					FIN SI
				FIN SI
		FIN POUR
	SI indexMin != x
		temp <- tableau[x]
		tableau[x] <- tableau[indexMin]
		tableau[indexMin] <- temp
	FIN SI
	FIN POUR
	RETOURNER tableau
FIN FONCTION
